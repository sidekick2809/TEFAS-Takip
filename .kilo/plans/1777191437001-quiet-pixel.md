# TEFAS API v2 Geçiş Planı

## Özet
TEFAS.gov.tr API'si güncellendi. Eski `/api/DB/*` endpoint'leri yeni `/api/funds/*` endpoint'leriyle değişti. Tarih formatı `yyyy-MM-dd`'den `yyyyMMdd`'e değişti. Bu plan, kod tabanındaki TEFAS API çağrılarının yeni sisteme geçişini sağlar.

## Değişiklik Detayları

### 1. Fon Fiyatları Çekme (BindHistoryInfo → fonGnlBlgSiraliGetir)
- **Eski Endpoint**: `POST /api/DB/BindHistoryInfo`
- **Yeni Endpoint**: `POST /api/funds/fonGnlBlgSiraliGetir`
- **Payload Değişikliği**:
  - Eski: `fontip=X&bastarih=DD.MM.YYYY&bittarih=DD.MM.YYYY&...`
  - Yeni: `{"fonTipi":"YAT","basTarih":"YYYYMMDD","bitTarih":"YYYYMMDD",...}` (JSON formatı)
- **Date Format**: `DD.MM.YYYY` → `YYYYMMDD`

### 2. Fon Geçmiş Getirileri (BindComparisonFundReturns → fonGetiriBazliBilgiGetir)
- **Eski Endpoint**: `POST /api/DB/BindComparisonFundReturns`
- **Yeni Endpoint**: `POST /api/funds/fonGetiriBazliBilgiGetir`
- **Payload Değişikliği**:
  - Eski: `calismatipi=2&fontip=YAT&bastarih=Başlangıç&...`
  - Yeni: `{"fonTipi":"YAT","calismaTipi":2,"donemGetiri1a":"1",...}` (JSON formatı)

### 3. Tarih Formatı
- Eski: `15.04.2026` (DD.MM.YYYY)
- Yeni: `20260415` (YYYYMMDD)

## Mevcut Kullanımlar

### Backend (server.js)
1. `/api/DB/BindHistoryInfo` - TEFAS proxy üzerinden fon fiyatları
2. `/api/DB/BindComparisonFundReturns` - TEFAS proxy üzerinden getiri karşılaştırması
3. `/api` prefix'li genel TEFAS proxy

### Frontend (main.js)
1. `fetchTefasData()` - 4 API çağrısı (3x BindHistoryInfo, 1x BindComparisonFundReturns)
2. `getTefasStatusData()` - 2 API çağrısı (BindComparisonFundReturns)
3. `getFormattedDates()` - Tarih formatlama

### Frontend (flow.js)
1. `fetchFlowData()` - 1 API çağrısı (BindHistoryInfo)
2. `getFlowDates()` - Tarih formatlama

## Çözüm Stratejisi

### Adım 1: Yeni Backend Endpoint'leri Oluşturma
Yeni TEFAS API'lerini kullanmak için server.js'e yeni endpoint'ler eklenmeli.

Eski proxy mantığı korunacak ancak yeni özel route'lar eklenecek:
- `POST /api/tefas/fon-gnl-blgsirali` - Fon fiyatları için
- `POST /api/tefas/fon-getiri-bazli` - Getiri karşılaştırması için

### Adım 2: Frontend API Çağrılarını Güncelleme
- `main.js`: Yeni endpoint'leri kullanacak şekilde güncellenecek
- `flow.js`: Yeni endpoint'leri kullanacak şekilde güncellenecek
- Tarih formatlama fonksiyonları güncellenecek

### Adım 3: Eski Proxy Devre Dışı Bırakma
Eski `/api/DB/*` endpoint'leri kaldırılamaz (geçiş dönemi uyumluluğu için) ancak kullanımdan çıkarılacak.

function formatTefasDate(isoDate) { // "2026-04-15" -> "20260415"
  const [y, m, d] = isoDate.split('-');
  return `${y}${m.padStart(2, '0')}${d.padStart(2, '0')}`;
}

## Riskler ve Çözümleri

### Risk 1: Yanıt Formatı Farklılığı
- **Risk**: Yeni API yanıt formatı farklı olabilir
- **Çözüm**: API yanıtı gelen veriyi eski formatla eşleşecek şekilde dönüştürme (adapter pattern)

### Risk 2: Tarih Formatı Karışıklığı
- **Risk**: Bazı yerlerde hâlâ eski format kullanılabilir
- **Çözüm**: Merkezi tarih formatlama fonksiyonu kullanımı

### Risk 3: Performans
- **Risk**: JSON parse/stringify ek yükü
- **Çözüm**: Negligible - modern JS engine'lar optimize edilmiştir
